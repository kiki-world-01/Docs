## Overview

Podium supports minting on-chain rewards on EVM chains (Base, Polygon, Arc Testnet) with a queued minting system via Privy server wallets. Rewards serve as verifiable, on-chain incentives with six distinct reward types.

## Endpoints

### Reward Queries

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/nft` | Query rewards by contract/owner |
| `GET` | `/user/{id}/nfts` | User's reward collection |
| `GET` | `/user/{id}/nfts/earned` | Rewards earned through programs |
| `GET` | `/user/{id}/nfts/redeemable` | Available for redemption |
| `GET` | `/user/{id}/nfts/redeemed` | Already redeemed |
| `GET` | `/user/{id}/nfts/{network}/{address}/{tokenId}/status` | Reward status |

### Reward Programs

| Method | Path | Description |
|--------|------|-------------|
| `POST` | `/nft-reward` | Create a reward program |
| `GET` | `/nft-reward/{id}` | Get reward details |
| `POST` | `/user/{id}/nfts/{network}/{address}/{tokenId}/redeem` | Redeem a reward |

### Airdrops

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/airdrop/{id}` | Airdrop details |
| `POST` | `/airdrop/{id}/delivered` | Mark airdrop as delivered |
| `GET` | `/airdrop/{id}/redemption` | Airdrop redemption tracking |

### Creator Collectibles

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/creator/{slug}/collectibles` | Public collectibles listing |
| `GET` | `/creator/{slug}/collectibles/metadata` | Collectible metadata |
| `GET` | `/creator/id/{creatorId}/collectibles` | Authenticated collectibles management |

## Reward Types

| Type | Description |
|------|-------------|
| `POINTS` | Redeem for Podium points |
| `FREE_PRODUCT` | Redeem for a specific product |
| `DISCOUNT_CODE` | Unlock a discount code |
| `EVENT_PASS` | Access to an event or experience |
| `CUSTOM` | Custom reward defined by creator |
| `EXCLUSIVE_ACCESS` | Gated content or feature access |

## Minting System

Rewards are minted on-chain through a queue system for reliability:

1. Mint request created → `QueuedMint` with status `PENDING`
2. Mint queue cron job processes pending mints
3. Privy server wallet executes the on-chain transaction
4. Status updated to `COMPLETED` or `ERROR`
