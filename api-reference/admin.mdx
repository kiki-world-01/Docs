## Overview

Admin endpoints manage platform-level configuration including API keys, organizations, subscription tiers, and the enrichment pipeline. Most require `ADMIN` role access.

<Note>
  Most developers interact with API keys and organization settings through the [Podium Developer Portal](https://www.podiumcommerce.xyz/onboarding). The admin API is for programmatic management and platform-level automation.
</Note>

## API Key Management

### Create a Key

<CodeGroup>

```bash cURL
curl -X POST https://api.podiumcommerce.xyz/api/v1/admin/api-keys \
  -H "Authorization: Bearer $PODIUM_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{ "organizationId": "clorg_abc" }'
```

```typescript SDK
import { createPodiumClient } from '@podium-sdk/node-sdk';
const client = createPodiumClient({ apiKey: process.env.PODIUM_API_KEY });
const key = await client.apiKeys.create({
  requestBody: { organizationId: 'clorg_abc' }
});
```

</CodeGroup>

```json
{
  "id": "clkey_xyz",
  "key": "podium_live_sk_abc123...",
  "organizationId": "clorg_abc",
  "isActive": true,
  "createdAt": "2026-03-07T12:00:00.000Z"
}
```

<Warning>
  The full API key is only returned once at creation time. Store it securely — subsequent queries return a masked version.
</Warning>

### Deactivate a Key

<CodeGroup>

```bash cURL
curl -X PATCH https://api.podiumcommerce.xyz/api/v1/admin/api-keys/clkey_xyz \
  -H "Authorization: Bearer $PODIUM_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{ "isActive": false }'
```

```typescript SDK
await client.apiKeys.update({
  id: 'clkey_xyz',
  requestBody: { isActive: false }
});
```

</CodeGroup>

Setting `isActive: false` immediately blocks all requests using this key.

### Rotate a Key

<CodeGroup>

```bash cURL
curl -X POST https://api.podiumcommerce.xyz/api/v1/admin/api-keys/clkey_xyz/rotate \
  -H "Authorization: Bearer $PODIUM_API_KEY"
```

```typescript SDK
const rotated = await client.apiKeys.createRotate({ id: 'clkey_xyz' });
```

</CodeGroup>

```json
{
  "id": "clkey_xyz",
  "key": "podium_live_sk_newkey456...",
  "previousKeyId": "clkey_xyz_old",
  "rotatedAt": "2026-03-07T12:00:00.000Z"
}
```

Key rotation:
1. Generates a new key value for the same key ID
2. Invalidates the old key value
3. Publishes an `api-key-changed` QStash event to flush Redis caches across all API instances
4. Returns the new full key (one-time display)

### Delete a Key

<CodeGroup>

```bash cURL
curl -X DELETE https://api.podiumcommerce.xyz/api/v1/admin/api-keys/clkey_xyz \
  -H "Authorization: Bearer $PODIUM_API_KEY"
```

```typescript SDK
await client.apiKeys.delete({ id: 'clkey_xyz' });
```

</CodeGroup>

Permanently removes the key. This cannot be undone.

### Create an Organization

<CodeGroup>

```bash cURL
curl -X POST https://api.podiumcommerce.xyz/api/v1/admin/organizations \
  -H "Authorization: Bearer $PODIUM_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Acme Beauty",
    "domain": "acmebeauty.com",
    "subscriptionTierId": "tier_growth"
  }'
```

```typescript SDK

  requestBody: {
    name: 'Acme Beauty',
    domain: 'acmebeauty.com',
    subscriptionTierId: 'tier_growth'
  }
});
```

</CodeGroup>

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | Yes | Organization display name |
| `domain` | string | No | Organization domain |
| `subscriptionTierId` | string | No | Subscription tier to assign |

```bash
curl -X PATCH https://api.podiumcommerce.xyz/api/v1/admin/campaigns/clcamp_xyz/status \
  -H "Authorization: Bearer $PODIUM_API_KEY" \

```

Valid transitions: `SUBMITTED` → `APPROVED` or `SUBMITTED` → `DRAFT` (rejection).

## Enrichment Pipeline

Admin endpoints for managing the [enrichment pipeline](/agentic/enrichment-pipeline).

### Trigger a Crawl

```bash
curl -X POST https://api.podiumcommerce.xyz/api/v1/admin/enrichment/run \
  -H "Authorization: Bearer $PODIUM_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{ "sourceId": "clsource_sephora" }'
```

Dispatches an `enrichment-crawl` QStash event for the specified source. The crawl fetches data via Firecrawl and queues extraction.

### Recompute Baselines

```bash
curl -X POST https://api.podiumcommerce.xyz/api/v1/admin/enrichment/baseline/recompute \
  -H "Authorization: Bearer $PODIUM_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{ "productId": "clprod_abc123" }'
```

Omit `productId` to recompute baselines for all products.

### Pipeline Status

```bash
curl https://api.podiumcommerce.xyz/api/v1/admin/enrichment/status \
  -H "Authorization: Bearer $PODIUM_API_KEY"
```

```json
{
  "sources": 12,
  "totalRecords": 45000,
  "pendingExtractions": 120,
  "pendingBaselines": 45,
  "lastCrawlAt": "2026-03-07T06:00:00.000Z",
  "lastBaselineAt": "2026-03-07T08:00:00.000Z"
}
```

## Task Administration

| Endpoint | Description |
|----------|-------------|
| `GET /admin/tasks/pending-review` | Tasks flagged for manual review |
| `POST /admin/tasks/{taskId}/resolve` | Manual approval/rejection |
| `POST /admin/tasks/{taskId}/retry-verification` | Retry failed oracle verification |

<CodeGroup>

```bash cURL
curl https://api.podiumcommerce.xyz/api/v1/admin/tasks/pending-review \
  -H "Authorization: Bearer $PODIUM_API_KEY"
```

```typescript SDK
const tasks = await client.admin.getTasksPendingReview();
```

</CodeGroup>

<CodeGroup>

```bash cURL
curl -X POST https://api.podiumcommerce.xyz/api/v1/admin/tasks/0xabc123.../resolve \
  -H "Authorization: Bearer $PODIUM_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{ "approved": true }'
```

```typescript SDK
await client.admin.resolveTaskManually({
  taskId: '0xabc123...',
  requestBody: { approved: true }
});
```

</CodeGroup>

See [Task Pool API](/api-reference/task-pool-api) for details.
